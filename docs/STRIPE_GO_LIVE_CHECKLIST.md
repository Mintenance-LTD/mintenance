## Stripe Dashboard setup

All of these require clicking in the [Stripe Dashboard](https://dashboard.stripe.com/)
**in both test mode and live mode**.

### 1. Enable Connect

Dashboard → **Connect** → **Get started** → choose **Platform or marketplace**.
Complete the platform profile (business info, URL, support contact, industry).

### 2. Enable Express accounts

Dashboard → **Connect** → **Settings** → **Express** → toggle ON.
Customize the branding (logo, primary color `#0D9488`, business name).

### 3. Enable payment methods (for homeowners)

Dashboard → **Settings** → **Payment methods**:

- [ ] **Card** — enabled by default
- [ ] **Bacs Direct Debit** — request activation (GB only, takes ~1 business day)
  - Requires UK business entity
  - Mandate URL auto-generated by Stripe
  - Customers see "Direct Debit" as a payment option in PaymentElement

### 4. Enable Stripe Tax

Dashboard → **Reporting** → **Tax forms** → **Get started**:

- [ ] Set platform's business tax ID (UK VAT number or UTR)
- [ ] Enable **1099-K** form generation (or equivalent UK reporting)
- [ ] Configure minimum thresholds if required by local regulation
- [ ] Stripe will auto-generate tax documents visible in each contractor's
      Express Dashboard at year-end

### 5. Configure webhook endpoint

Dashboard → **Developers** → **Webhooks** → **Add endpoint**:

- **Endpoint URL**: `https://mintenance.app/api/webhooks/stripe`
- **Events to send** (merge with existing events):
  - `account.updated` *(already handled)*
  - `setup_intent.succeeded` *(new)*
  - `setup_intent.setup_failed` *(new)*
  - `payment_method.detached` *(new)*
  - `transfer.created`
  - `transfer.failed`
  - `transfer.reversed`

Copy the webhook signing secret → set as `STRIPE_WEBHOOK_SECRET` in Vercel.

### 6. Disable rolling-reserve if enabled by default

Dashboard → **Balance** → **Payouts** → **Manage payout schedule**.
We pay contractors weekly on Fridays — confirm **Platform payouts** are set to
**daily** (so the platform balance is always available for our weekly transfers).

## Deployment sequence

1. Deploy backend to Vercel (no env var changes yet) — all new routes return
   501 or require DB tables that don't exist yet.
2. Run the migration: `npm run migrate:push` (or via Supabase dashboard).
3. Set `STRIPE_*` env vars in Vercel → redeploy.
4. Register webhook endpoint in Stripe Dashboard → copy new webhook secret.
5. Set `STRIPE_WEBHOOK_SECRET` → redeploy.
6. Test in **test mode** first:
   - Create a test contractor profile
   - Hit `/contractor/payouts/onboarding` → complete Stripe test onboarding
   - Use Stripe CLI to forward webhooks: `stripe listen --forward-to localhost:3000/api/webhooks/stripe`
   - Verify `profiles.stripe_payouts_enabled = true` after onboarding
   - Add test card via `/account/payment-methods/add` (test card: `4242 4242 4242 4242`)
   - Verify row in `payment_methods` table
7. Flip to live mode: swap `sk_test_` → `sk_live_`, `pk_test_` → `pk_live_`
8. Opt into accumulation mode when ready: set `ESCROW_USE_PAYOUT_ACCUMULATION=true`
9. First Friday after go-live, watch logs for `/api/cron/contractor-payouts`

## Verification queries

After onboarding a contractor:
```sql
SELECT
  id, email, role,
  stripe_connect_account_id,
  stripe_charges_enabled,
  stripe_payouts_enabled,
  stripe_transfers_active,
  stripe_requirements_pending
FROM profiles
WHERE role = 'contractor'
  AND stripe_connect_account_id IS NOT NULL
ORDER BY stripe_onboarding_completed_at DESC NULLS LAST
LIMIT 10;
```

After a weekly payout run:
```sql
SELECT
  c.first_name, c.last_name,
  t.amount_minor, t.currency, t.status, t.created_at
FROM contractor_payout_transfers t
JOIN profiles c ON c.id = t.contractor_id
WHERE t.created_at > NOW() - INTERVAL '7 days'
ORDER BY t.created_at DESC;
```

Contractor pending balances:
```sql
SELECT
  c.first_name, c.last_name,
  b.pending_amount_minor,
  b.lifetime_paid_out_minor,
  b.last_payout_at
FROM contractor_payout_balances b
JOIN profiles c ON c.id = b.contractor_id
ORDER BY b.pending_amount_minor DESC;
```

## Monitoring after go-live

Watch `security_events` + application logs for:
- `service: 'stripe-connect'` + `severity: 'CRITICAL'` → onboarding failures
- `service: 'payouts'` + `failed > 0` → transfer failures need investigation
- `service: 'stripe-webhook'` + `event_type: 'setup_intent.setup_failed'`
  → Elements flow friction
